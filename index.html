HTML (HyperText Markup Language)

  • Semantic HTML Elements = Name of Element describes its content and role on a webpage
  • Element and Tag used interchangably but Element is whole element from opening tag to closing tag, Tag is what's inside the brackets

  • The head element contains information/metadata about the webpage
    • Child Elements of Head will not be displayed
    • Common Head Elements include:
      • Style: defines CSS styles asso ciated with a website
      • Base: defines base URL for webpage
      • Link: connects external resource to HTML document
      • Meta: defines metadata (character set, description, keywords, author, and viewport)
      • Script: embeds client-side scripts in HTML document

  • The body element represents the visible content shown to the user
    • Common Body Elements Include:
      • Headings <h1, h2, h3 . . .>: Prominent titles and subtitles to organize a page
        • It is important for heading tags to be used in sequential order to enhance accesibility between devices
      • Paragraph <p>: represents a paragraph or block of text
      • Unordered List <ul>: Creates an unordered, bulleted list
      • Ordered List <ol>: Creates an numbered/lettered list
      • List Item <li>: A Child Element of lists that display the lists items
      • Images <img>: Contains info about images to be displayed on a webpage
        • Element will come with attributes to further define it. For example, "src" defines the location of an image
      • Line Break <br>: Creates a line of empty space between blocks of content
      • Anchor/Link <a>: Creates links to the same webpage or other webpages
        • Like Images, contains attributes to further define. "href" points to the url that is to be used for the link
    • The Header element displays information at the top of the webpage (often a navigation bar/intro content)
    • The footer element displays information at the bottom of a webpage ("I can code!)

CSS (Cascade Style Sheets)

  • CSS defines how webpage content defined by HTML is presented to user
    • Colors, Aesthetics, Visual Layout, Organization, etc.

  • CSS Rule Example: "div color: blue;
    • div: = Selector: defines the element or attributes to which the rules/declarations will apply
    • color:blue = Declaration: The rules that affect the element defined by the Selector, contains 2 components:
      • color = Property
      • blue = Value
  • Multiple Selectors can be combined in a single CSS Rule separated by commas, declarations will apply to all elements selected
  • Using an asterisk * as a selector will apply declarations to every element visible on the webpage
  • Refer to CSS BOX MODEL for example of how the Margin/Border/Padding/Content in an element are organized
    • This image was modified using inline CSS code
    • A margin indicates how much space we want around the outside of an element.
    • A padding indicates how much space we want around the content inside an element.
  • When defining the value of a color, you can refer to either the name of a color ("grey") or by its hexadecimal code/Hex # (#000000), the hex code will give a more precise hue
  • If using an external stylesheet, you must link the sheet to the webpage in the head section of your HTML code
  • You can create a class selector by starting it with a period. This will allow you to apply the class' CSS declarations to any HTML element assigned to it
    • Using this study guide as an example, the CSS class "card" linked to the HTML file from the stylesheet is applied to each <section> tag, so all the elements within those sections are affected by the "card" class.
  • The example code "margin: 40px auto;" assigns 2 values to the property
    • The first value (40px) will be assigned to the top and bottom of the element
    • The second value (auto) will be assigned to the left and right of the element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: Commits all files in the working branch (saves changes to working branch)
  • git commit -m: Commits staged changes from working branch to repository
    • "-m" flag associates a message with our commit that should be a short descriptive message of the changes implemented
  • Add vs Commit
    • git add: takes a modified file in your working directory and places the modified version in a staging area
    • git commit: takes everything from the staging area and makes a permanent snapshot of the current state of your repository that is associated with a unique identifier (-m)
      • This makes changes to the local repository, not the repo on GitHub
      • The Commit Message(-m) should describe the snapshot/changes made in the commit in a way that makes it easy for you or collaborators to understand what was changed and why and allows you to find/undo previous changes made if necessary
  • Push/Pull/Pull Request(PR)
    • Push:
    • Pull:
    • Pull Request (PR):
  • git pull origin main: Pulls remote "main" branch into the local "main" branch, syncing both online and local repositories
  • Save local changes to Remote Repository:
    1. git status double check current branch
    2. git add -A takes modified files in working directory and places it in the staging area
    3. git commit -m Commits staged changes to remote repository with descriptive message
    4. git pull origin main Ensures local branch is up to date before pushing changes
    5. git push origin [branch-name] Pushes commited changes in current branch to remote repo

JavaScript

  • JavaScript os a high-level dynamic programming language that allows users to interact with websites
  • As in CSS, /* defines a comment
  • A Variable is a named container that allows us to store data in our code and can be defined as "const", "let", or more commonly "var"
  • Control Flow is the order in which a computer executes a code in script. This goes top to bottom.
  • Variables can be given values that have different Data Types
    • Strings: A sequence of text declared as (x = 'String')
    • Numbers: A numerical value, needs no quotes (x = 4)
    • Boolean: A true/false value, needs no quotes (x = true)
    • Array: A structure that allows you to store multple values in a single reference, contained by brackets and separated by commas (x = [red, green, blue])
      • Refer to each member of an array by calling their Array Index represented by sequenced numbers starting with "0" that indicates each items place in the order of the array [0,1,2,3...]
        • In the above example, the color green would be represented by x[1] because it is the second value in the array
      • Arrays can hold values of different data types. For example, a students name, age, and enrollement status are contained in one Array, represented as (var studentInfo = ['John', 24, true])
    • Object: "This can be anything. Everything in JavaScript is an object and can be stored in a variable. Keep this in mind as you learn." (x = document.querySelector('h1'))
  • Operators are mathematical symbols that produce a result based on two values
    • Addition (+): Adds 2 Numbers or Strings Together (1 + 2; 'Hello' + 'World!')
    • Subtraction, Multiplication, Division (-,*,/): Performs expected mathematical function (2-1; 2*4; 4/2;
    • Assignment (=):Assigns a value to a variable (let myVariable = 'x';)
    • Strict Equality (===):Performs a test to see if two values are equal, returns a boolean result (let myVariable = 'x'; myVariable === 'y';)
    • Not, Does Not Equal (!,!==):Returns the logically opposite value of what it precedes, turns a 'true' into a 'false'
  • Conditionals: Code structures used to test if an expression returns true or not. The most common example is an if/then statement
  • Functions: Ways of packaging functionality you wish to reuse so as to avoid repeating the same code. Functions can be recalled in code when necessary like "myFunction()"
    • Functions will not run unless they are called somewhere in the script.
  • Messages can be sent in a browsers console/DevTools with the method console.log which can be used to help developers in the debugging process
  • JavaScript Files should be linked to HTML files at the end of the Body section with a Link tag, similar to how a CSS file is linked
  • For Loop: Uses a predictable pattern of indices to perform a task on all items in an array by executing a single block of code repeatedly. This loop must be interrupted with another statement such as an if statement to execute a block of code a specific number of times. Contains 3 Important Expressions:
    • Initialization: Expression/Variable Declaration evaluated once before loop begins. Newly declared variables with 'var' are not local to the loop, decalred variables using 'let' are local to the loop
    • Condition: Expression to be evaluated before each loop iteration. If (expression === true){statement is executed}; else{execution exits loop and goes to first statement after 'for' loop}
    • Afterthought: Expression to be evaluated at the end of each loop interation; occurs before next evaluation of 'condition', generally used to update/increment the counter variable (x + 1 is a common expression here which can be more simply written as x++)
    • Statement: Code that is executed as long as the condition evaluates as true
    • All 3 Expressions in the for loop are not required but included as a standard
  • Blocks of code can be "commented out" like this: /* CODE BLOCK */
  • "Math" is a JavaScript Object that has properties/methods used for mathematical constants and functions.
    • For example, "Math.random()" generates a random number from 0 to <1
